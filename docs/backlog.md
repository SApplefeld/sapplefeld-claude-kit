# Backlog

The living handoff and next-steps doc for the kit. It carries active items only. When an item is done, it moves out to a dated snapshot in `archive/` (`backlog-YYYY-QN.md`) rather than being struck through in place, so this file stays lean and readable.

Per-plan history does not live here. A plan's Chapters are its own append-only record and travel with the plan into `archive/` when it closes. This file is for cross-effort next-steps that do not belong to any single open plan.

- **Explore channels for unattended interactive runs (research preview).** A channel is an MCP server that injects external events into the interactive session as model input (code.claude.com/docs/en/channels-reference). Two uses for long runs, both keeping the work on subscription billing: a local webhook or scheduler nudging an overnight run's next turn, and permission relay (v2.1.81+) answering tool-approval prompts from a phone. A channel cannot execute `/resume` (events are conversation input, not UI commands), so it complements compact-session's interactive handoff rather than replacing it. Caveats: custom channels need `--dangerously-load-development-channels` at every session start while off the allowlist, senders must be gated (an ungated channel is a prompt-injection vector), and undelivered events drop silently. Decided 2026-07-08: prototype when the next overnight-run need materializes. The need is materializing: the 2026-07-10 overnight chain ran on manual dispatch because the resume relay was absent (see machine repairs).

- **Relay request queue: single-file last-writer-wins collision (2026-07-16).** `request.txt` is a machine-global single file, so two boundary requests written inside one 10-second poll interval mean the later `mv` silently replaces the earlier, and the losing session's resume never fires (its closing report's manual `/resume` line is the recovery). Negligible at today's cadence (runs compact every couple of hours against a 10-second window), but it is the one cross-run interference between two concurrently relayed terminal windows. Sketch when it earns fixing: writers emit unique `request-<destination-uuid>.txt` names, the watcher globs oldest-first and archives per file. Both sides must land together (an old watcher ignores the new names), and the change needs a live typing retest on an armed machine, which is why it is parked here rather than ridden along with the 2026-07-16 kit-goal binding round.

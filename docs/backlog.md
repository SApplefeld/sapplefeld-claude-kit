# Backlog

The living handoff and next-steps doc for the kit. It carries active items only. When an item is done, it moves out to a dated snapshot in `archive/` (`backlog-YYYY-QN.md`) rather than being struck through in place, so this file stays lean and readable.

Per-plan history does not live here. A plan's Chapters are its own append-only record and travel with the plan into `archive/` when it closes. This file is for cross-effort next-steps that do not belong to any single open plan.

- **Open experiment: Fable advisor on execution sessions (2026-07-09; adopted as default same day).** Whether a lower-model execution session plus `/advisor fable` matches Fable-led orchestration quality at lower cost (Anthropic reports Sonnet + Opus-advisor beating Sonnet solo on SWE-bench Multilingual at lower cost). The fable advisor is now the standing default on below-fable sessions (`Fable Spend: none (cost hold)` is the off-switch), so the protocol is monitoring rather than alternation: the Chapter `Metrics` line records advisor state per section; compare advisor-on sections against the pre-adoption baseline and any cost-hold runs on review rounds, NEEDS_CONTEXT, and escalations, and revert the default with one line in `brainstorming`/`executing-work` if the quality delta does not cover the consultation spend. Verified facts this rides on (live-probed on v2.1.205, 2026-07-09): the advisor is fully active in headless `claude -p` (an `advisorModel` in settings suffices; the `--advisor` flag also works though absent from `--help`); every subagent inherits it with no per-agent override; a subagent whose pinned model outranks the advisor silently drops it; one consultation roughly quadrupled a small turn's cost (uncached transcript re-read at advisor rates). Watch for: implementers resolving spec gaps via the advisor instead of NEEDS_CONTEXT (the guardrail's RED case), and advisor spend appearing on unattended headless runs that never named it. Data point 2026-07-11: consultations bill out of band and never appear in transcripts, so the transcript study cannot measure them; the bound is ~$2.30 per consultation at 200k context and ~$5.75 at 500k (an uncached full-transcript read at Fable input rates), an order of magnitude under Fable-led execution, and the Opus-main-plus-advisor posture was adopted as the strict rule on that basis.
- **Chain-vs-subagents benchmark data point (2026-07-09).** First live end-to-end run of chain mode (Linux container, Bun, v2.1.205): a 4-section synthetic plan run three ways, all 12 section gates green, engine compactions succeeded and compacted successors resumed with working context. Numbers (Sonnet everywhere, advisor off): fresh-session-per-section $0.44 / 63s model time; chain worker implementing inline with boundary compaction $0.42 / 53s (plus two uncounted summarizer passes); chain worker dispatching a subagent per section $1.02 / 169s. Read: at small section size, fresh-vs-chain-inline is a wash (orientation cost and carried-context cost cancel), compaction held the chain's context growth nearly flat, and per-section delegation cost 2.3x because each subagent's ~25k-token boot dwarfed the tiny work - confirming the "too small to brief goes inline" carve-out. The deciding variable is section churn relative to the fixed per-agent boot cost, which the tier system already encodes; re-test at real section scale before drawing a stronger conclusion. Harness lives in the 2026-07-09 session scratchpad (not committed). Real-scale data point 2026-07-11 (transcript study): chain workers ran $0.23-0.41 per main-chain call vs ~$0.38 for a monolithic orchestrator - no per-call economy; the chain's real win is the forced per-section context reset, which the 200k check gate now gives interactive sessions too, and pending Agent SDK metering makes chain-worker turns API-billed while in-session subagents stay on subscription. Posture recorded in the operating-model spec: chain is for unattended stretches only.

- **Explore channels for unattended interactive runs (research preview).** A channel is an MCP server that injects external events into the interactive session as model input (code.claude.com/docs/en/channels-reference). Two uses for long runs, both keeping the work on subscription billing: a local webhook or scheduler nudging an overnight run's next turn, and permission relay (v2.1.81+) answering tool-approval prompts from a phone. A channel cannot execute `/resume` (events are conversation input, not UI commands), so it complements compact-session's interactive handoff rather than replacing it. Caveats: custom channels need `--dangerously-load-development-channels` at every session start while off the allowlist, senders must be gated (an ungated channel is a prompt-injection vector), and undelivered events drop silently. Decided 2026-07-08: prototype when the next overnight-run need materializes.
